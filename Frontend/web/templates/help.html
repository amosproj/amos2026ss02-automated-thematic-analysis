Getting Started

The Automated Thematic Analysis tool simplifies the qualitative research process into three main steps:

  1. Upload your data: Head over to the Transcripts page to securely upload your interview texts. Optionally, attach metadata via the Demographic Data section.
  2. Generate a Codebook: On the Codebooks page, you can instruct our LLM to automatically generate a hierarchical theme-structure from your corpus, or you can build/upload one manually.
  3. Run the Analysis: Navigate to the Analysis workspace to apply your Codebook across all documents. The system will extract specific quotes and link them to themes for easy review.

Frequently Asked Questions

By default, we connect to the NHR@FAU cluster, which runs state-of-the-art open-weights models (such as Qwen or Mixtral) in a secure, privacy-compliant environment. You can change the active provider on the Home page.

Your transcripts are stored securely in our database and are only processed by the selected LLM provider. When using the default FAU NHR provider, your data never leaves the university's secure network infrastructure.

This tool is entirely open-source. If you wish to experiment with different prompt strategies or integrate new generation methods, the backend services are highly compartmentalized. Check out the project repository linked on the right.

You can seamlessly add support for alternative LLM providers (e.g., OpenAI, Anthropic, or any OpenAI-compatible API) by modifying a few configuration files.

  1. Define the Provider: Open Backend/app/llm/providers.py and add a new ProviderSpec entry to the PROVIDERS tuple. Define its id, label, and the environment variable names it will look for (e.g., api_key_attr="LLM_API_KEY_CUSTOM").
  2. Manage API Keys Safely: Open your local Backend/.env file (or copy it from .env.example). Add your API key matching the variable name you defined in step 1. Because .env is excluded via .gitignore, your keys remain securely on your machine and are never committed to the repository.
  3. Code Communications: Because the system dynamically reads the provider registry, simply registering it in providers.py is usually enough! However, if your provider requires specific custom headers or non-standard client initialization, you should modify the build_chat_model() function inside Backend/app/llm/client.py.
  4. Restart & Select: Once configured, restart the backend server. Your new provider will instantly appear as an option in the "LLM Provider" dropdown on the Home page.
